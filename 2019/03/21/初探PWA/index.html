背景

近年来，虽然Web应用迎来了爆发式的发展，但是在移动端远远没有达到桌面设备的流行程度。尽管前端的许多技术和设计在应用性能上得到了很大的提升和突破，但是Web应用始终突破不了Web平台本身的一些限制：加载网页需要下载的延迟、依赖浏览器作为应用入口等。

尽管其实Web站和APP功能差不多，用户还是依然会去选择使用app，用户觉得APP不会出现白屏、并且功能入口都在桌面上。而web应用则是一个应用中的应用，而且加载的速度很慢。这个时候，PWA出现了，PWA的出现就是来对标原生应用的。

PWA是什么

PWA全称叫 Progressive Web App 即 渐进式Web应用，什么叫渐进式？我的理解是你可以用它的一部分或者全部，或者全部不用，但是不会影响应用正常运行，也就是做到了“全浏览器支持”。我们可以大致总结为啥用户更加偏向于使用原生应用而不是网站：

  • 可靠
  • 启动快
  • 可见的主屏幕图标
  • 离线可使用
  • 推送可将用户带回应用

那么PWA能给我们带来些什么？可以看看官方的描述：

简单总结一下就是：及时加载、快速响应、沉浸式体验。

Service Worker

PWA应用之所以能够做到离线使用，靠的就是Service Worker，它可以选择性的缓存网站的部分资源以提供离线的使用。
Service Worker其实是在后台运行的Worker脚本，使用JS编写，我们可以用来拦截网络请求，处理推送消息和执行其他的许多任务。

Service Worker 机制

Service Worker 就像是我们的应用和服务器之间的一个中间商，它可以全权控制网站发出的每一个请求。Service Worker有以下几个特点：

  • 运行在自己的全局脚本上下文中
  • 不绑定到具体网页
  • 无法修改网页中的元素，因为它无法访问DOM
  • 只能使用HTTPS

因为Service Worker运行在worker上下文中，所以无法访问DOM，它与应用的主要JavaScript运行在不同的线程，所以不会被阻塞，是完全异步的。就像图中所示，Service Worker 可以全权控制应用中所有进出的网络请求。

Service Worker 生命周期


当用户第一次导航到一个URL上时，服务器会返回响应网页，注册的时候会调用register()函数，此时Service Worker开始下载、解析并且执行。如果在这个期间出现任何错误，register()返回的Promise都会执行reject操作，然后Service Worker就会被废掉。一旦Service Worker成功执行了，安装事件就会激活(service worker是基于事件的)，意思就是可以进入所有事件中的任何一个。安装完成之后，Service Worker就会激活，如果生命周期内所有阶段都成功了，那么Service Worker便准备就绪了，待使用。
当第一次加载页面的时候，Service Worker还没有激活，所以它不会处理任何请求，也就是说只有刷新页面，Service Worker逻辑才会启动。

App Shell 模型

Web应用与原生应用给人第一感觉的差距就在于原生应用哪怕在没有网络的情况下，依然不会出现完全白屏的情况，而是展示出一个大概的框架，只是空出动态的内容部分，并且还会给出Loading的提示。但是Web应用一旦离开网络，要不是完全白屏，要不就是小恐龙界面，这两种用户体验是差距非常大的。原生应用是纯天然支持App Shell模型的，所以我们可以在第一次进入网站的时候，把应用的主题框架先缓存起来，在无网络的情况下，使用缓存的资源，这样就达到了和原生应用一样的效果了。

PWA观感

PWA应用除了能够利用Service Worker来达到各种骚操作之外，还可以让Web应用长得更像原生应用，其中包含将站点添加到主屏幕和各种定制化。

Web应用清单

Web应用清单就是一个简单的JSON配置文件，可以通过配置这个JSON文件来定制应用的各种信息，包括应用名称、作者信息、应用描述、应用图标等等。

manifist.json

  • name： 提示用户安装应用时出现的文本
  • short_name：安装后主屏幕应用的文本
  • start_url：从主屏幕开启web应用所出现的第一个页面。

  • dispaly：应用如何向用户展示，分别为fullscreen、standalone、minimal-ui和browser。

    -fullscreen: 全屏显示，会尽可能将所有的显示区域都占满；
    -standalone: 独立应用模式，这种模式下打开的应用有自己的启动图标，并且不会有浏览器的地址栏。因此看起来更像一个Native App；
    -minimal-ui: 与standalone相比，该模式会多出地址栏；
    -browser: 一般来说，会和正常使用浏览器打开样式一致。

  • theme_color：对浏览器地址栏进行着色

  • icons：添加到主屏幕上时应用显示的图标

    保存到桌面后再启动，会有一个类似于原生应用的开屏图，是由background_color + icons 组成

IOS(Safiri)中的处理方式

  • apple-mobile-web-app-title 应用的标题

  • apple-touch-icon: 应用的图标
